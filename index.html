Features:

  • Create an infinite number of profiles, each with an infinite number of services, each of which with an infinite number of accounts.
  • Cross-platform, coded in C# and fully functional in both .NET and Mono. Invium will run and behave in the exact same way across the following operating systems: Windows, Linux, MacOS.
  • Strong cryptography technologies involved (AES 256-bit, SHA-512 and PBKDF2 key derivation) ensure cracking the data is going to be time consuming and very costly.
  • Support for on-the-fly language switching.
  • Very intuitive, easy to use interface.
  • Incredibly small; Invium itself is a single-file product. One executable. Not counting the data file that is created when you start using it; the data file that holds your actual information.

What Invium is NOT

  • Invium is not as feature rich as some other password managers you can find online.
    • It will not integrate with your webbrowser.
    • There is no app for it for any mobile device.
    • It does not sync anything with any form of cloud.
      • For the time being, I would suggest BitTorrent Sync to start syncing Invium between multiple PCs.
  • Invium is not a substitute to common sense.
    • Do not access your most sensitive and key accounts from public PCs or essentially any PC that you do not own; it simply is common sense not to trust that which you do not own, who knows what's been installed?
    • No amount of precautionary measures on my part will ever cover every possible contingency; I cannot possibly predict or code against every possible loophole or security leak, particularly not those left by poor PC management. In short: Ensure your PC is clean of virusses or any other form of malware/spyware and keep it that way.

Getting Invium

Getting Invium is very simple.

  1. Go to this page
  2. Download the version appropriate to your OS and architecture. There are 32-bit and 64-bit downloads for Windows, Linux and MacOS.
  3. Place the downloaded executable in a folder of your chosing; there is no installation procedure involved.
  4. (Optional) Create a shortcut for it in whatever way you feel comfortable with.
  5. Start using it, you're done!

Compiling Invium

For those interested in a more hands-on approach, compiling Invium should be relatively straightforward.

  1. From this page either clone the project using Git or just download the .ZIP.
  2. Open the solution (Invium.sln) in your favourite IDE (it has been compiled successfully in both Visual Studio Express as well as MonoDevelop).
  3. From the configurations menu (where you are able to select which configuration to use for the build) pick whichever platform you are building it for.
    • .NET or Mono (both available in AnyCPU and x64)
  4. Build it and you should be able to find the executable in the Invium/bin subfolder of wherever you cloned/extracted Invium to.
  5. Copy that executable to wherever you want and run it.
  6. (Optional) Create a shortcut for it in whatever way you feel comfortable with.
  7. Start using it, you're done!

Contributing to the development of Invium

Want to help develop Invium? If so, feel free to fork the project and start coding and hitting me with pull requests. Personally I would recommend Linux + MonoDevelop for the development. As far as the coding style is concerned; it is based on on-the-fly code formatting by MonoDevelop in conjunction with some tweaked StyleCop settings and it will be enforced. So expect any accepted pull request to go through some formatting changes.

I could specifically use a hand with the translations. The infrastructure is there for on-the-fly language switching. See Language.cs in the project for information on how to add translations.
